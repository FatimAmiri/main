Basics

By installing COMPAS (see Installation), the following packages are installed:

  • compas

  • compas_blender

  • compas_ghpython

  • compas_plotters

  • compas_rhino

The compas package provides the core functionality of the COMPAS framework. The packages compas_rhino, compas_ghpython, and compas_blender provide integration in Rhino, Grasshopper (through GHPython) and Blender, respectively. compas_plotters provides 2D visualization tools based on matplotlib.

Code structure

In all packages, functionality is made available at the first sub-package level.

from compas.datastructures import Mesh
from compas.geometry import Point
from compas.numerical import dr_numpy

# Rhino only

from compas_rhino.artists import MeshArtist
from compas_rhino.geometry import RhinoSurface

# Blender only

from compas_blender.artists import MeshArtist

Core functionality

The core functionallity of the framework is written entirely in CPython such that it can easily be used across platform, in various coding environments, and inside software tools the provide a Python scripting interface.

To speed up computation, some functionality is based on Numpy/Scipy. These functions are suffixed with _numpy.

They are available in all environments except for Rhino and GHPython since Numpy and Scipy are not available for IronPython. _numpy functions can be used in Rhino or Grasshopper through Remote Procedure Calls (compas.rpc)

CAD integration

compas can be used directly in CAD software that provides support for Python scripting. The CAD packages (compas_rhino, compas_ghpython, compas_blender) provide classes and functionality for visualizing COMPAS data structures and geometry, for processing native CAD geometry, and for building simple UIs.

Visualization is handled by Artists. An artist is available for every data structure, geometric primitive and shape.

from compas.datastructures import Mesh
from compas_rhino.artists import MeshArtist
